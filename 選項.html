vimwiki有全域與區域(每個維基檔案)選項可供調整。

全域選項

所有的全域選項的設定都使用下列這個格式：

let g:option_name=option_value

所有的區域(每個維基檔案)選項在維基列表中，是一種字典的配對形式。 詳細內容參考|g:vimwiki_list|

g:vimwiki_list vimwiki-multiple-wikies 每個在g:vimwiki_list中的項目是 is a |Dictionary| that holds all customization available for a wiki represented by that item. It is in form of > {'option1': 'value1', 'option2: 'value2', ...}

看下列的範例:

let g:vimwiki_list = [{'path': '~/my_site/', 'path_html': '~/public_html/'}]

它將一個維基指定到目錄 ~/my_site/ 下，且html檔案生成至目錄 ~/public_html/ 下。

看下面一個範例:

let g:vimwiki_list = [{'path': '~/my_site/', 'path_html': '~/public_html/'}, \ {'path': '~/my_docs/', 'ext': '.mdox'}]

我們指定兩個維基，第一個維基跟上個範例一樣。第二個維基被指定到目錄 ~/my_docs/ 下，且它的檔案延伸檔名以 .mdox 結尾。

如果要清除g:vimwiki_list，需將維基指定為預設的選項：

let g:vimwiki_list = [{}, \ {'path': '~/my_docs/', 'ext': '.mdox'}]

你也可以用一個分割的字典建立維基。

let wiki_1 = {} let wiki_1.path = '~/my_docs/' let wiki_1.html_header = '~/public_html/header.tpl' let wiki_1.nested_syntaxes = {'python': 'python', 'c++': 'cpp'}
let wiki_2 = {} let wiki_2.path = '~/project_docs/' let wiki_2.index = 'main'
let g:vimwiki_list = [wiki_1, wiki_2]

PER WIKI OPTIONS viwmiki-local-options

vimwiki-option-path

Key 預設 value~ path ~/vimwiki/

描述~ Wiki files location: > let g:vimwiki_list = [{'path': '~/my_site/'}] <

vimwiki-option-path_html

Key 預設 value~ path_html ~/vimwiki_html/

描述~ HTML files converted from wiki files location: > let g:vimwiki_list = [{'path': '~/my_site/',

\ 'path_html': '~/my_site_html/'}]

If you omit this option path_html would be path - '/' + '_html/': > let g:vimwiki_list = [{'path': '~/okidoki/'}]

ie, path_html = '~/okidoki_html/'

vimwiki-option-auto_export

Key 預設 value 值s~ auto_export 0 0, 1

描述~ Set this option to 1 to automatically generate HTML file when corresponding wiki page is saved: > let g:vimwiki_list = [{'path': '~/my_site/', 'auto_export': 1}]

This will keep you HTML files up to date.

vimwiki-option-index

Key 預設 value~ index index

描述~ Name of wiki index file: > let g:vimwiki_list = [{'path': '~/my_site/', 'index': 'main'}]

NOTE: Do not add extension.

vimwiki-option-ext

Key 預設 value~ ext .wiki

描述~ Extension of wiki files: > let g:vimwiki_list = [{'path': '~/my_site/',

\ 'index': 'main', 'ext': '.document'}]

< vimwiki-option-syntax

Key 預設 value 值s~ syntax default default, media

描述~ Wiki syntax. You can use different markup languages (currently default vimwiki and MediaWiki) but only vimwiki's default markup could be converted to HTML at the moment. To use MediaWiki's wiki markup: > let g:vimwiki_list = [{'path': '~/my_site/', 'syntax': 'media'}] <

vimwiki-option-html_header

Key 預設 value~ html_header

描述~ Set up file name for html header template: > let g:vimwiki_list = [{'path': '~/my_site/',

\ 'html_header': '~/public_html/header.tpl'}]

This header.tpl could look like: >

<html> <head> <link rel="Stylesheet" type="text/css" href="%root_path%style.css" /> <title>%title%</title> <meta http-equiv="Content-Type" content="text/html; charset=utf-8" /> </head> <body>

where %title% is replaced by a wiki page name or by a |vimwiki-title| %root_path% is replaced by a count of ../ for pages buried in subdirs:

if you have wikilink dir1/dir2/dir3/my page in a subdir then %root_path% is replaced by '../../../'.

vimwiki-option-html_footer

Key 預設 value~ html_footer

描述~ Set up file name for html footer template: > let g:vimwiki_list = [{'path': '~/my_site/',

\ 'html_footer': '~/public_html/footer.tpl'}]

This footer.tpl could look like: >

</body> </html>

<

vimwiki-option-css_name

Key 預設 value~ css_name style.css

描述~ Set up css file name: > let g:vimwiki_list = [{'path': '~/my_pages/',

\ 'css_name': 'main.css'}]

< or even > let g:vimwiki_list = [{'path': '~/my_pages/',

\ 'css_name': 'css/main.css'}]

<

vimwiki-option-maxhi

Key 預設 value 值s~ maxhi 1 0, 1

描述~ Non-existent wiki links highlighting could be quite slow and if you don't want it set maxhi to 0: > let g:vimwiki_list = [{'path': '~/my_site/', 'maxhi': 0}]

This disables filesystem checks for wiki links.

vimwiki-option-nested_syntaxes

Key 預設 value 值s~ nested_syntaxes {} pairs of highlight keyword and vim filetype

描述~ You can make preformatted text to be highlighted with a different syntaxes available for vim. For example the following setup in your vimrc: > let wiki = {} let wiki.path = '~/my_wiki/' let wiki.nested_syntaxes = {'python': 'python', 'c++': 'cpp'} let g:vimwiki_list = [wiki]

would give you python and c++ highlighting in: > 

 for i in range(1, 5):
     print(i)

 #include "helloworld.h"
 int helloworld()
 {
    printf("hello world");
 }

or in: > 

 #include "helloworld.h"
 int helloworld()
 {
    printf("hello world");
 }

 for i in range(1, 5):
     print(i)

vimwiki-option-diary_rel_path

Key 預設 value~ diary_rel_path diary/

描述~ Related to |vimwiki-option-path| path for diary wiki-files.

vimwiki-option-diary_index

Key 預設 value~ diary_index diary

描述~ Name of wiki-file that holds all links to dated wiki-files.

vimwiki-option-diary_header

Key 預設 value~ diary_header Diary

描述~ Name of the header in |vimwiki-option-diary_index| where links to dated wiki-files are located.

vimwiki-option-diary_link_count

Key 預設 value~ diary_link_count 4

描述~ Number of maximum dated links placed on one line. Ex:

Diary

2010-01-30 2010-01-29 2010-01-28 2010-01-27
2010-01-26 2010-01-25

GLOBAL OPTIONS viwmiki-global-options

Use: >

let g:option_name=option_value

to set them.

g:vimwiki_hl_headers

Highlight headers with =Reddish=, ==Greenish==, ===Blueish=== colors.

值 描述~ 1 Use predefined colors to highlight different header levels. 0 Use |hl-Title| or VimwikiHeader1-VimwikiHeader6 (if defined

in a colorscheme)

預設: 1

g:vimwiki_hl_cb_checked

Checked list items could be highlighted with a color:

  • the whole line could be highlighted with the option set to 1.
  • I wish vim could use strikethru.

值 描述~ 1 Highlight checked [X] check box with |group-name| "Comment". 0 Don't.

預設: 0

g:vimwiki_global_ext vimwiki-temporary-wiki

If a file with a registered wiki extension is opened in a dir that is not listed in |g:vimwiki_list| then:

值 描述~ 1 make a temporary wiki in that dir. 0 don't make temporary wiki it that dir.

預設: 1

g:vimwiki_upper g:vimwiki_lower

This affects WikiWord detection. By default WikiWord detection uses English and Russian letters. You can set up your own: > let g:vimwiki_upper = "A-Z\u0410-\u042f" let g:vimwiki_lower = "a-z\u0430-\u044f"

g:vimwiki_auto_checkbox

if on, creates checkbox while toggling list item.

值 描述~ 0 Do not create checkbox. 1 Create checkbox.

預設: 1

Ex: Press <C-Space> (|:VimwikiToggleListItem|) on a list item without checkbox to create it: >

  • List item

result: >

  • List item

g:vimwiki_menu

GUI menu of available wikies to select.

值 描述~ '' No menu 'Vimwiki' Top level menu "Vimwiki" 'Plugin.Vimwiki' "Vimwiki" submenu of top level menu "Plugin" etc.

預設: 'Vimwiki'

g:vimwiki_stripsym

Change strip symbol -- in Windows you cannot use /*?<>:" in file names so vimwiki replaces them with neutral symbol (_ is default): >

let g:vimwiki_stripsym = '_'

You can change it to a <space> for example: >

let g:vimwiki_stripsym = ' '

g:vimwiki_badsyms

Consider you do not like spaces in filenames (as some vimwiki users do). In that case you can set up bad symbols that would be converted to |g:vimwiki_stripsym|: >

let g:vimwiki_badsyms = ' '

Now files for all links with spaces would be created like 'links_with_spaces'.

This option is a complement one to |g:vimwiki_stripsym|.

g:vimwiki_listsyms

String of 5 symbols for list items with checkboxes. 預設 value is ' .oOX'.

g:vimwiki_listsyms[0] is for 0% done items. g:vimwiki_listsyms[4] is for 100% done items.

g:vimwiki_use_mouse

Use local mouse mappings from |vimwiki-local-mappings|.

值 描述~ 0 Do not use mouse mappings. 1 Use mouse mappings.

預設: 0

g:vimwiki_folding

Enable/disable vimwiki's folding/outline. Folding in vimwiki is using 'expr' foldmethod which is very flexible but really slow.

值 描述~ 0 Disable folding. 1 Enable folding.

預設: 0

g:vimwiki_fold_lists

Enable/disable folding of list subitems.

值 描述~ 0 Disable list subitem's folding. 1 Enable list subitem's folding.

預設: 0

g:vimwiki_fold_trailing_empty_lines

Fold or do not fold empty lines between folded headers.

值 描述~ 0 Fold only one empty line. The rest empty lines are unfolded. 1 Fold in all empty lines.

預設: 0

g:vimwiki_camel_case

If you do not want WikiWord to be a link this setting is just for you.

值 描述~ 0 Do not make links from CamelCased words. 1 Make links from CamelCased words.

預設: 1

g:vimwiki_list_ignore_newline

This is HTML related. Convert newlines to
s in multiline list items.

值 描述~ 0 Newlines in a list item are converted to
s. 1 Ignore newlines.

預設: 1

g:vimwiki_use_calendar

Create new or open existing diary wiki-file for the date selected in Calendar. See |vimwiki-calendar|.

值 描述~ 0 Do not use calendar. 1 Use calendar.

預設: 1

g:vimwiki_browsers VimwikiWeblinkHandler

You can open external weblinks in a webbrowser. Webbrowsers are listed in |g:vimwiki_browsers|.

For win32 it is: chrome, opera, firefox and explorer. For other OSes it is: opera, firefox and konqueror.

The first available browser from the list is used to open weblink. If you have opera and firefox and want weblinks to be opened in the latter just: > let g:vimwiki_browsers=['C:\Program Files\Firefox\firefox.exe']

or redefine VimwikiWeblinkHandler function: > function! VimwikiWeblinkHandler(weblink)

let browser = 'C:\Program Files\Firefox\firefox.exe' execute '!start "'.browser.'" ' . a:weblink

endfunction

g:vimwiki_table_auto_fmt

Turn on/off table auto-formatting.

值 描述~ 0 Do not auto-format tables. 1 Auto-format tables.

預設: 1

g:vimwiki_w32_dir_enc

Convert directory name from current |encoding| into 'g:vimwiki_w32_dir_enc' before it is created.

If you have 'enc=utf-8' and set up >

let g:vimwiki_w32_dir_enc = 'cp1251'

< then following the next link with <CR>: >

郈?邽赲迮?/邾邽?

> would convert utf-8 '郈?邽赲迮?' to cp1251 and create directory with that name.

預設: ''

g:vimwiki_CJK_length

Use special method to calculate correct length of the strings with double wide characters. (To align table cells properly)

值 描述~ 0 Do not use it. 1 Use it.

預設: 0

Note: Vim73 has new function |strdisplaywidth|, so for Vim73 users this option is obsolete.

g:vimwiki_dir_link

This option is about what to do with links to directories -- directory/, papers/, etc.

值 描述~ '' Open 'directory/' using standard netrw plugin. 'index' Open 'directory/index.wiki', create if needed. 'main' Open 'directory/main.wiki', create if needed. etc.

預設: '' (empty string)

g:vimwiki_html_header_numbering

Set this option if you want headers to be auto numbered in html.

ex: >

1 Header1 1.1 Header2 1.2 Header2 1.2.1 Header3 1.2.2 Header3 1.3 Header2 2 Header1 3 Header1

etc.

值 描述~ 0 Header numbering is off. 1 Header numbering is on. Headers are numbered starting from

header level 1.

2 Header numbering is on. Headers are numbered starting from

header level 2.

etc. Example when g:vimwiki_html_header_numbering = 2: >

Header1 1 Header2 2 Header2 2.1 Header3 2.1.1 Header4 2.1.2 Header4 2.2 Header3 3 Header2 4 Header2

etc.

預設: 0

g:vimwiki_html_header_numbering_sym

|g:vimwiki_html_header_numbering|的結束符號。

值 描述~ '.' 點會被加到標題的數字的最後面。 ')' 結束小括號會被加到標題的數字的最後面。 等等。

使用選項：

let g:vimwiki_html_header_numbering = '.'

標題將看起來像：

  1. Header1 1.1. Header2 1.2. Header2 1.2.1. Header3 1.2.2. Header3 1.3. Header2
  2. Header1
  3. Header1

預設: '' (空)

g:vimwiki_file_exts

延伸檔名列表以逗號分隔。

假設你有下面的連結: [[my_script.php][my script]]。 假如g:vimwiki_file_exts選項中，有設定'php'延伸檔名。這個連結在html文件中，將被生成為 <a href="my_script.php">my script</a> 。 否則它將被生成為 <a href="my_script.php.html">my script</a> (筆記 .html)。

預設: 'pdf,txt,doc,rtf,xls,php,zip,rar,7z,html,gz'

g:vimwiki_valid_html_tags

能在vimwiki中，使用的html標記列表以逗號分隔。

預設: 'b,i,s,u,sub,sup,kbd,br,hr'

g:vimwiki_conceallevel

In vim73 |conceallevel| is local to window, thus if you open viwmiki buffer in a new tab or window, it would be set to default value.

Vimwiki sets |conceallevel| to g:vimwiki_conceallevel everytime vimwiki buffer is entered.

預設: 3
