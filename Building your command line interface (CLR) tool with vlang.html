Building your command line interface (CLR) tool with vlang on Windows

When I am a student, I like Linux course. The teacher teaches us how to type commands in a black screen. There is no colorful icons. Just boring text and black screen. We have to login into a remote Linux server in a terminal . Trying many commands. Afterward I know Linux equals these commands. Although there is a Window, the teacher doesn't teach that. I think it's so simple, a mouse can easily use it.

Today I will make my mkdir command line interface tool with vlang. The command style likes GNU mkdir command. I will reference the common module. It contains many useful utility for building a GUI like's command. My mkdir command named md has been distinguished from the built-in mkdir command. Merely creating a folder is boring. The folder name can contains emoji. If I type and execute the Github emoji shortcode on the command line, it will convert this shortcode to an emoji unicode text. For example, the command is md :stuck_out_tongue:_Hello. After executed, it will create a folder named 😛_Hello.

Prepare vlang environment

Download v_windows from V Programming Language. Extract the zip file. Go to the directory v_windows. Use the command v up updates source code to latest version. If it doesn't work, clone source code from the vlang repositoy. Afterward execute make.bat.

Create directory

Create our project folder using command mkdir md. Change the directory to the directory cd md. Initial our project via executing the command v int. It creates a md.v file automatically. This is our main source code file. Downland common.v to current directory.

*The emoji character 😛 is utf8 encoding. The code file md.v must save as utf8 file encoding. Otherwise the emoji character will be changed to the question mark character. *Let's add some code to md.v. We need to import os module to use mkdir function. And import common module we mentioned before. Then define constant name and description of the program. The Settings structure contains verbose and dirnames variables. When we use the command with option verbose, it will print "created directory '??'" on the command line. The message tells us that we just created a directory. If we want to a respond, the verbose option is useful. The value of the option verbose will store to the Settings structure. Another variable is dirnames. We can create multiple directories at the same time. Using the command md aa bb cc. It will create directory aa, bb and cc. So we need a string array to store these directory name. 

module main

import os
import common

const (
	app_name        = 'md'
	app_description = 'Create the DIRECTORY(ies), if they do not already exist. If directory name contains Github emoji shortcode, it will convert to a emoji UTF8 code.
'
)

struct Settings {
	verbose 	bool
	dirnames           []string
}

In the main function I call the args function. It will return a Settings structure data. The data passed into md function. 

fn main() {
	md(args())
}

We create directories and replace emoji string in the md function. If there is option verbose, print a message. 

fn md(settings Settings) {
	mut dirnames := settings.dirnames

	for dirname in dirnames {
		new_dirname:=dirname.replace(':stuck_out_tongue:','😛')
		os.mkdir(new_dirname) or {
			eprintln('$app_name: cannot create directory \'$dirname\': File exists ')
			continue
		}
		if settings.verbose {println('$app_name: created directory \'$dirname\'')}
	}
}

The args function handles the command options and parameters. 


fn args() Settings {
	mut fp := common.flag_parser(os.args)
	fp.application(app_name)
	fp.description(app_description)

	verbose := fp.bool('verbose', `v`, false, 'print a message for each created directory')

	dirnames := fp.finalize() or {
		eprintln(err)
		println(fp.usage())
		exit(1)
	}
	if dirnames.len==0 {
		eprintln('$app_name: missing operand')
		eprintln("Try '$app_name --help' for more information.")
		exit(1)
	}
	return Settings{verbose,dirnames}
}

We have source code. Let's run it with the command v.. It generates a md.exe file. Now we can use the command to create a directory with an emoji directory name. Type the command on the command line.

./md.exe :stuck_out_tongue:_keep_smile
