Create a Math Quiz Web Application with vlang on Windows

What is vlang?

The V Programming Language. Simple, fast, safe, compiled. For developing maintainable software.

I will use the Vweb module to create the math quiz web application on Windows. It can accept query string http://localhost:8080/add/operand1/operand2.

Prepare vlang environment

Download v_windows from V Programming Language. Extract the zip file. Go to the directory v_windows. Use the command v up updates source code to latest version. If it doesn't work, clone source code from the vlang repositoy. Afterward execute make.bat.

Initial Project

Create our project folder using command mkdir quiz. Change the directory to the folder cd quiz. Initial our project via executing the command v int. It creates a quiz.v file automatically. This is our main source code file. Open it to see what's inside. 

module main

fn main() {
	println('Hello World!')
}

There is a main function. Like C programming language). It is the entry point of the program. First line declares the module name as main. The default module name is main. A module likes a folder. The folder contains many functions inside. How to use the module? For example, we want to use a say_hi function. It is written in myfile.v file. The say_hi function has to use pub keyword to declare the function that is a public function.Then we can use the say hi function in main.c. 

// Declare module name as mymodule
module mymodule

// To export a function we have to use `pub`
pub fn say_hi() {
	println('hi from mymodule!')
}

import mymodule

fn main() {
        // Call function with module name. It is very clear that the say_hi function comes from the module mymodule.
	mymodule.say_hi()
}

Create a Vweb project

I want to use the Vweb module to build my web application. So I must import the module to the quiz.v file. Add declare an app variable in the main function. Afterward we can use functions inside the app variable. My code will become like this. 

module main

import vweb

struct App {
	vweb.Context
}

fn main() {
	app := App{}
	vweb.run(app, 8081)
}
// router path
['/index']
pub fn (mut app App) index() vweb.Result {
	return app.text('Hello world from vweb!')
}

The command prompt output: 

[Vweb] Running app on http://localhost:8081/

We can see our web application via the url http://localhost:8081/. Or use the link http://localhost:8081/index to see the same web application.

How to impliment a Vweb query string?

Query string is part of URL. URL links a page or document. Using query string can pass many different values to generate many different page. It's really helpful. In JavaScript you can get parems using searchParams property of the url objet. 

const params = new URL('https://example.com?foo=1&bar=2').searchParams;
params.get('foo'); // "1"
params.get('bar'); // "2"

In vlang the query string is different. It's not a key value pair format just a value. I think this format prevents people from guessing. What does value mean? <mark>Remember that the router name , function name and html template file name must be the same. </mark> 

// Declare a query string
['/add/:operand1/:operand2'; get]
pub fn (mut app App) add(operand1 int, operand2 int) vweb.Result {
	// Print debug information in command prompt
	println(operand1)
	println(operand2)
	// Declare and initial a immutable variable sum. If you want to modify the value of the variable using `mut` keyword at the beginning of the line.
	sum:=operand1+operand2
	// In this function we have three variables operand1, operand2 and sum. These will pass to the html templat file.
	// Find a html file name as add.html
	return $vweb.html()
}

Create the add.html template file. Use at @ character to get the value from the variable. Afterward execute command v run quiz.v. 

<body>
<p>
@operand1 + @operand2 = ? <span id="sum" style="display:none">@sum</span>
</p>
<button onclick="ans()">Show Answer</button>
<script>
function ans(){
document.getElementById('sum').style.display="block";	
}
</script>
</body>

Now we can add two numbers from the query string http://localhost:8081/add/12/18.
